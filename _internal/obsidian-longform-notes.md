# Obsidian и Longform

## Obsidian

Obsidian используется как среда мышления и письма:

- сцены;
- журнал;
- персонажи;
- локации;
- ссылки между заметками;
- быстрый доступ к методологии.

## Longform

Longform можно использовать для порядка сцен и сборки рукописи.

Рекомендуемый принцип:

- художественный текст — в `02-manuscript/scenes/`;
- служебная информация — во frontmatter, комментариях или отдельных карточках;
- итоговая сборка — в `04-output/`.

## Исключение служебной информации

Для служебных пометок внутри сцены можно использовать:

```markdown
%%
Заметка для автора или ИИ-агента.
Не должна попасть в итоговый текст.
%%
```

Для структурированных данных:

```yaml
---
scene_type: action
pov: ""
goal: ""
conflict: ""
outcome: ""
---
```

При сборке рукописи нужно удалять frontmatter и комментарии.

## Риск синхронизации

Не стоит одновременно использовать много автоматических синхронизаций:

- Obsidian Sync;
- Dropbox/iCloud/Google Drive;
- Git auto-sync;
- редактирование Longform с разных устройств.

Лучше выбрать один основной механизм синхронизации.
