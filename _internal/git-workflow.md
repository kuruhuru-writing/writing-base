# Git-процесс

## Для `writing-base`

Коммиты должны фиксировать изменения метода, шаблонов и инструкций.

Примеры:

```bash
git add .
git commit -m "Добавил шаблоны сцен по Суэйну"
git commit -m "Уточнил протокол работы ИИ-агента"
git commit -m "Обновил чек-лист непрерывности"
```

## Для репозитория романа

Коммиты должны фиксировать смысловые этапы работы.

Примеры:

```bash
git commit -m "Заполнил компас романа"
git commit -m "Наметил основные маяки"
git commit -m "Написал сцену открытия"
git commit -m "Обновил promises после первой главы"
git commit -m "Пересобрал план второй главы"
```

## Ветки

Для экспериментов:

```bash
git checkout -b darker-ending
git checkout -b alternative-midpoint
git checkout -b new-protagonist-motivation
```

Правило:

> Ветка нужна, когда хочется попробовать идею, которая может сильно изменить роман.

## Когда делать commit

- после новой сцены;
- после изменения компаса;
- после изменения маяков;
- после принятия важного решения;
- после пересборки главы;
- перед крупным экспериментом.

## Что не коммитить

- временные экспорты;
- локальные настройки workspace;
- случайные файлы;
- приватные материалы, не относящиеся к проекту.
