# Обзор методологии

## Назначение

Эта система помогает писать роман гибридным способом: часть заранее продумывается, часть открывается во время письма.

Цель — сохранить одновременно:

- творческую свободу;
- причинность сюжета;
- управляемость большого текста;
- возможность регулярно консультироваться с ИИ-агентом;
- историю решений через Git.

## Основная формула

```text
Компас → Маяки → Сцены → Реакции → Журнал → Проверка → Пересборка
```

## Уровни работы

### 1. Общая картина

Файлы:

- `compass.md`
- `beacons.md`
- `characters/`
- `world/`

Задача: понимать, куда движется роман.

### 2. Ближайший участок

Файлы:

- текущие сцены;
- журнал;
- promises/questions/decisions.

Задача: понимать, что писать дальше.

### 3. Сцена

Сцена действия:

```text
цель → препятствие/конфликт → осложнение
```

Сцена реакции:

```text
эмоция → осмысление → дилемма → решение → новая цель
```

### 4. Контроль состояния

После каждой значимой сцены нужно обновлять:

- что изменилось;
- какие вопросы открылись;
- какие обещания даны читателю;
- какие решения приняты;
- что может быть следующей сценой.

## Главный принцип

Структура не должна заменять творческое решение. Она должна помогать автору удерживать роман в собранном состоянии.
