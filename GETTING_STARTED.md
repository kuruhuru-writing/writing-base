# Быстрый старт

## 1. Что такое `writing-base`

Это база твоей писательской системы. Здесь живут общие принципы, шаблоны и инструкции, которые можно использовать во всех романах.

`writing-base` отвечает на вопрос:

> Как я вообще пишу романы?

Конкретный роман отвечает на другой вопрос:

> Что происходит именно в этом романе?

Поэтому конкретные романы лучше выносить в отдельные репозитории.

## 2. Минимальный порядок работы

### Сначала развиваешь систему

1. Читаешь `methodology/00-overview.md`.
2. Изучаешь `methodology/01-hybrid-novel-method.md`.
3. При необходимости правишь методологию под свой стиль.
4. Обновляешь шаблоны в `templates/`.

### Потом создаёшь роман

1. Создаёшь новый GitHub-репозиторий, например `novel-working-title`.
2. Копируешь туда содержимое `project-blueprints/novel-project/`.
3. Открываешь новый репозиторий как отдельный Obsidian vault.
4. Заполняешь `01-planning/compass.md`.
5. Заполняешь `01-planning/beacons.md`.
6. Начинаешь писать сцены в `02-manuscript/scenes/`.
7. Ведёшь журнал в `03-journal/`.

## 3. Как работать с ИИ-агентом

Для полноценной консультации агенту нужно дать не только рукопись, но и состояние проекта.

Минимальный пакет контекста:

```text
00-method/hybrid-novel-method.md
01-planning/compass.md
01-planning/beacons.md
01-planning/promises.md
01-planning/questions.md
01-planning/decisions.md
02-manuscript/scenes/<текущие сцены>
03-journal/writing-log.md
```

Если нужно проверить конкретную сцену, добавь:

```text
checklists/scene-checklist.md
_internal/agent-operating-protocol.md
```

Если нужно решить, что писать дальше, добавь:

```text
agent-prompts/plan-next-scene.md
agent-prompts/chapter-plan.md
```

## 4. Как обновлять `writing-base`

Когда ты улучшаешь методологию, фиксируй изменения в Git.

Примеры commit-сообщений:

```bash
git commit -m "Уточнил метод сцены по Суэйну"
git commit -m "Добавил шаблон реакции после сцены"
git commit -m "Обновил протокол работы ИИ-агента"
```

## 5. Главный принцип

`writing-base` должен оставаться чистым, универсальным и пригодным для повторного использования.

Если материал относится только к одному роману — ему место в репозитории этого романа.
